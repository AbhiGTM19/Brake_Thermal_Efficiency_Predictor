About This Project

This project delivers a full-stack machine learning solution to predict Brake Thermal Efficiency (BTE) of Internal Combustion Engines. BTE is a critical performance metric that reflects how effectively an engine converts chemical energy from fuel into mechanical energy.

Developed as part of an MTech MLOps-focused curriculum, the application integrates core components of the MLOps lifecycle—data preprocessing, model training, experiment tracking, model registry, containerization, and CI/CD-friendly deployment using Docker and Render. The objective is not only to provide accurate predictions but also demonstrate production-grade reproducibility and deployment workflows.

Key Features

Ridge Regression Model

Leverages L2 regularization to reduce overfitting and multicollinearity for stable and robust predictions of BTE.

Interactive UI

Simple and intuitive form-driven interface for real-time inference of engine efficiency metrics.

Asynchronous Prediction

Built with AJAX-based form submission to provide fast responses without full page reloads.

Experiment Tracking with MLflow

All training runs are logged with parameters, metrics, and artifacts using MLflow, enabling easy comparison and version control.

Dockerized Workflow

Packaged into a cross-platform Docker image (linux/amd64) for consistent deployment on Render or any cloud-native platform.

Cloud Deployment

Deployed on Render Web Services with container image pulled directly from Docker Hub.

The Prediction Model

The backend ML model is built using Ridge Regression and trained on a curated dataset of experimental engine configurations and corresponding BTE values.

Training Pipeline Highlights:

  • StandardScaler: Applied to normalize feature ranges and improve convergence.
  • GridSearchCV: Used for hyperparameter tuning (alpha selection) with 5-fold cross-validation to optimize performance.
  • MLflow: Handles model lifecycle—including experiment tracking, metric logging, model serialization, and versioning.
  • Pickle: Final model and scaler are serialized as .pkl files for inference within the app.

Each run is logged with metadata, and the best-performing model is versioned and registered for reproducible deployment.
